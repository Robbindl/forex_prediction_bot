STEP 3 — OPTIONAL API KEYS (.env)
════════════════════════════════════════════════════════════

All keys are optional. Components degrade gracefully without them.

  NEWSAPI_KEY=your_key            # broader news coverage
                                  # free at https://newsapi.org

  TWITTER_BEARER_TOKEN=your_token # already in your .env from Phase 1
                                  # Twitter tracker uses it automatically

  REDDIT_CLIENT_ID=your_id        # optional — raises Reddit rate limit
  REDDIT_CLIENT_SECRET=your_secret

Without any keys:
  - NewsAnalyzer:       polls CryptoPanic (free, no key)
  - RedditTopicTracker: polls public Reddit JSON (no key)
  - TwitterTopicTracker: disabled with a single log line

════════════════════════════════════════════════════════════
TROUBLESHOOTING
════════════════════════════════════════════════════════════

Problem: "[NewsAnalyzer] CryptoPanic fetch: 403"
CryptoPanic occasionally blocks anonymous requests. The analyzer
will retry on the next poll cycle (5 minutes). No action needed.

Problem: "[RedditTracker] r/CryptoCurrency error: 429"
Reddit rate-limited. The 5-second sleep between subreddits usually
prevents this. If it persists, increase POLL_INTERVAL_PER_SUB from
600 to 900 in reddit_topic_tracker.py.

Problem: "[TwitterTracker] Rate limited — backing off 15 minutes"
Expected on Basic tier — only 1 search/15 min. The backoff is
automatic. If you need more volume, upgrade to Basic+ on Twitter.
