TELEGRAM OUTPUT EXPLAINED
════════════════════════════════════════════════════════════

🤖 META AI    regime=trending_bull  ensemble=0.84
              tech=40%  sent=25%  whale=15%  flow=12%  macro=8%
              conf 0.90 → 0.93  (+0.03 boost)

regime        — current market condition detected by MarketConditionClassifier
ensemble      — weighted average of all 5 engine scores (0.0–1.0)
              0.5 = neutral, 1.0 = max bullish, 0.0 = max bearish
tech/sent/... — how much weight each engine got this signal
conf x → y   — confidence before and after Meta AI adjustment

════════════════════════════════════════════════════════════
ADJUSTING WEIGHTS (OPTIONAL)
════════════════════════════════════════════════════════════

If you find the default weights don't suit your trading style,
override them at runtime — no file editing needed:

    from ml.meta_model import weighter

    # Give technical more weight in ranging markets
    weighter.set_override("ranging", {
        "technical":  0.35,
        "sentiment":  0.15,
        "whale":      0.20,
        "orderflow":  0.20,
        "macro":      0.10,
    })

Or edit ml/meta_model/model_weighting_engine.py REGIME_WEIGHTS
directly and restart the bot.

════════════════════════════════════════════════════════════
ADDING FUTURE SIGNAL ENGINES
════════════════════════════════════════════════════════════

Phase 7, 8, or any new signal source can add its score to
the ensemble by adding one method to ensemble_predictor.py:

    @staticmethod
    def _my_new_engine_score(signal, context):
        score = context.get("my_new_score")
        if score is None:
            return None
        return float(score)   # return 0.0–1.0

Then add it to _collect_scores():

    "my_engine": self._my_new_engine_score(signal, context),

Add the weight to REGIME_WEIGHTS in model_weighting_engine.py
and it automatically appears in Telegram and the ensemble.

════════════════════════════════════════════════════════════
TROUBLESHOOTING
════════════════════════════════════════════════════════════

Problem: "[Pipeline] Loaded 7 layers" (not 8)
Fix: Make sure layer8_meta_ai.py is in your layers/ folder
     and pipeline.py has been replaced with the Phase 6 version.

Problem: Meta AI always shows regime=ranging
Fix: Phase 1 data feeds (funding, OI) may not be running.
     Check logs for "[FundingMonitor] Started" and "[OIMonitor] Started".
     Regime detection still works from price data alone — ranging is
     the safe default when funding/OI data is unavailable.

Problem: ensemble=0.500 on every signal (neutral)
Fix: Phase 3 order flow imbalance not reaching Layer 3.
     Check "[OrderFlow] Subscribed to ORDER_BOOK_UPDATE" in logs.
     Without order book data, orderflow score defaults to None
     and is excluded from the ensemble.

Problem: TestModelWeightingEngine::test_weights_sum_to_one FAILED
Fix: You edited REGIME_WEIGHTS and the weights no longer sum to 1.0.
     The engine normalises them automatically but the test checks
     the raw values. Make sure each regime sums to exactly 1.0.
