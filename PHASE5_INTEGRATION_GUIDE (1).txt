Drop run_lab.py in your project root and run:

    python run_lab.py

You will see an interactive menu:

  3.  Optimise parameters for a strategy
  4.  Test one strategy across multiple assets
  5.  Full report — all presets on all your assets
  6.  Backtest a custom strategy config

STEP 5 — ACTIVATING A STRATEGY FOR LIVE TRADING
════════════════════════════════════════════════════════════

After running run_lab.py you will see output like:

  golden_cross           Sharpe=2.1  WinRate=83%  PnL=+$3,400  Trades=6
  triple_ema             Sharpe=1.8  WinRate=72%  PnL=+$2,100  Trades=18
  macd_rsi_confluence    Sharpe=1.2  WinRate=61%  PnL=+$890    Trades=22
  rsi_scalper            Sharpe=0.3  WinRate=48%  PnL=+$180    Trades=45

You decide which to activate. Say you choose golden_cross and triple_ema.

Open strategy_lab/live_bridge.py and find LIVE_STRATEGY_CONFIGS.
It looks like this by default (all commented out):

    LIVE_STRATEGY_CONFIGS: List[Dict] = [
        # StrategyBuilder.triple_ema_config(),
        # StrategyBuilder.golden_cross_config(),
        # StrategyBuilder.macd_rsi_confluence_config(),
        # StrategyBuilder.adx_ema_momentum_config(),
        # StrategyBuilder.bollinger_rsi_reversion_config(),
    ]

Uncomment the ones you want:

    LIVE_STRATEGY_CONFIGS: List[Dict] = [
        StrategyBuilder.golden_cross_config(),
        StrategyBuilder.triple_ema_config(),
    ]

Save the file. Restart bot.py. You will see in the logs:

    [Voting] Live bridge: added 'golden_cross'
    [Voting] Live bridge: added 'triple_ema'
    [bot] Phase 5 live strategies active: ['golden_cross', 'triple_ema']

VotingStrategy now has 5 voters:
    RSIStrategy       → vote
    MACDStrategy      → vote
    BollingerStrategy → vote
    golden_cross      → vote   ← from lab
    triple_ema        → vote   ← from lab

To deactivate a strategy — comment it out again and restart.

STEP 6 — ADDING A CUSTOM STRATEGY CONFIG
════════════════════════════════════════════════════════════

You are not limited to the 15 presets. Write any config using
the supported indicators and operators.

Supported indicators:
    rsi, ema, macd, bollinger, atr, volume_ma, stoch, adx

Supported operators:
    >  <  >=  <=  cross_above  cross_below

Example — Stochastic + EMA trend filter:

    from strategy_lab import run_backtest

    my_config = {
        "name":    "my_stoch_strategy",
        "version": "1.0",
        "indicators": [
            {"name": "stoch", "params": {"k_period": 14, "d_period": 3}},
            {"name": "ema",   "params": {"period": 50}},
            {"name": "atr",   "params": {"period": 14}},
        ],
        "entry_rules": [
            {"col": "stoch_k", "op": "cross_above", "col2": "stoch_d", "direction": "BUY"},
            {"col": "stoch_k", "op": "<",  "val": 50},
            {"col": "close",   "op": ">",  "col2": "ema_50"},
        ],
        "confidence_boosts": [
            {"col": "stoch_k", "below": 30, "boost": 0.07},
        ],
        "stop_mult": 1.5,
        "tp_mult":   2.5,
    }

    result = run_backtest(my_config, "BTC-USD", "crypto")
    print(result.summary())

If happy with results, add it directly to LIVE_STRATEGY_CONFIGS:

    LIVE_STRATEGY_CONFIGS: List[Dict] = [
        my_config,
    ]

════════════════════════════════════════════════════════════
TROUBLESHOOTING
════════════════════════════════════════════════════════════

Problem: "No OHLCV data available for BTC-USD"
Fix: Check yfinance is working:
     python -c "import yfinance as yf; print(yf.download('BTC-USD', period='5d'))"

Problem: run_lab.py shows all strategies with 0 trades
Fix: Increase periods in run_backtest() — default is 500 bars.
     Some strategies like golden_cross need 200+ bars to warm up.

Problem: "[Voting] Live bridge load error: ..."
Fix: Check that strategy_lab/ folder is in your project root.
     Run: python -c "from strategy_lab.live_bridge import LIVE_STRATEGY_CONFIGS"

Problem: Strategy added to LIVE_STRATEGY_CONFIGS but not appearing in logs
Fix: Make sure you saved live_bridge.py and fully restarted bot.py.
     The strategies are loaded fresh on each restart.

Problem: Bot making too many trades after adding lab strategies
Fix: Increase min_votes in VotingStrategy (default is 2).
     More voters = harder to get majority = fewer but higher quality trades.
     Edit strategies/voting.py: VotingStrategy(min_votes=3)
