════════════════════════════════════════════════════════════
STEP 2 — INSTALL NEW DEPENDENCY
════════════════════════════════════════════════════════════

Open a terminal in your project folder and run:

  pip install websocket-client

(Everything else — requests, redis — is already in your
requirements.txt.)

Add to requirements.txt:
  websocket-client>=1.8.0

════════════════════════════════════════════════════════════
STEP 3 — WIRE INTO bot.py
════════════════════════════════════════════════════════════

Open bot.py and find the section labelled:
  # ── Portfolio risk engine ───────────────────────────────

Add the following block DIRECTLY AFTER the AutoTrainer block
(around line 210 of bot.py):

    # ── Phase 1 — Institutional data feeds ───────────────────────────────────
    try:
        from services.data_ingestion import start_all as start_data_feeds
        start_data_feeds(exchanges=["binance", "bybit"])
        logger.info("[bot] Phase 1 data feeds started")
    except Exception as e:
        logger.warning(f"[bot] Phase 1 data feeds failed to start: {e}")

That is the ONLY change to bot.py.

════════════════════════════════════════════════════════════
STEP 4 — OPTIONAL: add FRED_API_KEY to your .env
════════════════════════════════════════════════════════════

The macro collector is disabled silently if the key is absent.
To enable it, add to your .env file:

  FRED_API_KEY=your_fred_key_here

Get a free key at: https://fred.stlouisfed.org/docs/api/api_key.html

════════════════════════════════════════════════════════════
TROUBLESHOOTING
════════════════════════════════════════════════════════════

Problem: "ModuleNotFoundError: websocket"
Fix:     pip install websocket-client

Problem: "[ExStream] binance error: [SSL: CERTIFICATE_VERIFY_FAILED]"
Fix:     Run in an admin terminal:
         pip install --upgrade certifi

Problem: "[ExStream] bybit WS error: Connection refused"
Fix:     Bybit may be geoblocked in your region — use a VPN
         or remove "bybit" from the exchanges list:
         start_data_feeds(exchanges=["binance"])

Problem: Redis events not appearing
Fix:     Confirm Redis is running:
         redis-cli ping   → should return PONG
         If not: start Redis with  redis-server
